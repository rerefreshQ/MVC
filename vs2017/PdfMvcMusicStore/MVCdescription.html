ASP.NET MVC 应用程序结构(PDF-P39)

目录 用途
/Controllers 处理 URL 请求的 Controller 类
/Models 表示和操纵数据以及业务对象的类
/Views 负责呈现输出结果(如HTML)的 UI 模板文件
/Script JavaScript 库文件和脚本(.js)
/fonts Bootstrap 模板系统包含的一些自定义 Web 字体
/Content CSS、图像和其他站点内容，而非脚本
/App_Data 存储想要读取/写入的数据文件
/App_Start 功能的配置代码，如路由、捆绑和 Web API

建立及配置 MusicStore 项目(PDF-P37)

建立 StoreController 里面需要3个方法： index 索引页面列出商店里包含的音乐类型
Browse 列出所有音乐专辑的页面
Details 列出有关该专辑所有信息的页面

控制器中的参数--PDF52

Browse 方法修改为：检索从 URL 传过来的查询字符串值 http://localhost:28136/store/Browse?genre=xx
Details 方法修改为：使其读取和显示一个名为ID的输入参数。
这里不像前面的方法那样把 ID 值作为一个查询字符串参数，而是将 ID 值直接嵌入到 URL 中。
ASP.NET MVC 默认路由约定，如果操作方法中有名为 ID 的参数，那么 APS.NET MVC 会自动将这个 URL 片段作为参数传递过来。 		http://localhost:28136/store/Details/5

在其他一些应用中，我们可能需要指定完全位于不同目录结构中的视图。针对这种情况我们可以使用~符号的语法来提供视图的完整路径：

{
return View("~/Views/Example/Index.cshtml");
